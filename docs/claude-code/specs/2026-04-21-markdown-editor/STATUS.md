## Decisions carried forward (read before resuming)

- **Execution mode:** `superpowers:subagent-driven-development` with a pragmatic cadence — full three-subagent flow (implementer + spec review + code quality review) for tasks with real code or architectural touches; inline-by-controller for verbatim config / YAML / single-flag fixes.
- **Architecture override to PLAN.md:** `main.rs` stays a thin caller; the Tauri `Builder` lives in `lib.rs::run()`. All plan text that says "rewrite `main.rs`" means "update `lib.rs::run()`". See the Phase 1 preamble added in commit `6141a6b`.
- **bigint vs number:** ts-rs 8 emits `mtime_ms: bigint` in the generated `src/lib/ipc/types.ts`. At runtime the wire value is a plain JSON number. Phase 3 IPC wrappers coerce at the boundary and re-export cleaned types with `number`. See the Phase 3 preamble added in commit `d0bc1cc`.
- **Stray ts-rs output:** `src-tauri/src/lib/ipc/types.ts` is gitignored (prevents re-commit on `cargo test`).
- **`@types/react` / `@types/react-dom`** pinned to `^18` to match runtime.
- **`vitest run --passWithNoTests`** — required so CI stays green until the first TS test lands in Phase 3.
- **`"csp": null`** in `tauri.conf.json` — deferred hardening for Phase 8 when the markdown renderer lands.
- **Phase 2 deviations from PLAN.md:**
  - Plan's test asserted the emitted path string-equal to the subscribed path. On macOS `notify-debouncer-full` emits `/private/var/...` canonicalized paths while `TempDir` returns `/var/...`. Fixed by canonicalizing both sides of the compare in `src-tauri/tests/watcher_basic.rs` — semantics of the test unchanged.
  - Plan's `watcher.rs` omitted `use notify::Watcher as _;`. Required because the `notify::Watcher` trait provides the `.watch(path, mode)` method on the inner debouncer's watcher, and it conflicts by name with our `Watcher` struct (aliased to `_` to import the trait for method resolution only).
  - Plan's `lib.rs` imports said `use tauri::{Emitter, Manager};`. In Tauri 2, `App::handle()` is a direct method, so `Manager` was unused. Kept only `use tauri::Emitter;` to avoid warnings.
  - Plan's new `pub fn run()` body did not include `.plugin(tauri_plugin_opener::init())`. Followed the plan verbatim and dropped it from the builder chain. The `tauri-plugin-opener` dep still lives in `src-tauri/Cargo.toml` — a later phase can reintroduce it if a feature needs it.
- **Not verified in Phase 2 checkpoint:** `pnpm tauri dev` was not launched (it opens a native window). Manual verification recommended before starting Phase 3 frontend work.
